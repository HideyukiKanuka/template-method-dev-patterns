DEV-TM-003: コンストラクタ内テンプレートメソッド呼び出し

B: 誤適用（Misapplication） 逸脱タイプ: 実装 根拠: 4件 合議: agree
逸脱パターン名コンストラクタ内テンプレートメソッド呼び出し
対象デザインパターンTemplate Method
逸脱タイプ実装
A/B/C 判定B: 誤適用（Misapplication）
一般形式AbstractClass のコンストラクタ内で TemplateMethod() または abstract な PrimitiveOperation() を呼び出し、ConcreteClass のフィールドが未初期化のまま実行される
症状と結果Java のオブジェクト初期化順序（スーパークラスコンストラクタ → サブクラスフィールド初期化）により、サブクラスのフィールドが初期化される前に PrimitiveOperation() が実行される。結果として null 参照、0 など初期値の誤使用、不正な状態での処理が発生する。
解決策TemplateMethod() の呼び出しをコンストラクタから分離し、クライアントが明示的に初期化メソッドを呼ぶ形（Two-step initialization）に変更する。または Factory Method / Builder パターンで初期化完了後に処理を開始する。
防止策Effective Java（Bloch）の指針「コンストラクタは overridable メソッドを呼んではいけない」を徹底する。静的解析ツール（PMD、SpotBugs）で constructor 内の overridable 呼び出しを検出する。
逸脱例abstract class Vehicle のコンストラクタが calcMileage() を呼ぶ。ConcreteClass Car の fuel フィールドがまだ初期化されていないため calcMileage() が 0 を返す。
